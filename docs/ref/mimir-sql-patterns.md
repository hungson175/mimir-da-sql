## Proven SQL Patterns

### Moni MAU
```sql
SELECT COUNT(DISTINCT AGENT_ID)
FROM `momovn-prod.MBI_DA.SEMANTIC_QLCT_VISIT`
WHERE PRODUCT = 'MONI'
  AND MONTH = '2025-12-01'    -- format: YYYY-MM-01
  AND ACTION = 'CHAT'
```
**Result:** 308,931 (Dec 2025)

### Total MoMo Transactions
```sql
SELECT SUM(NUMBER_TRANS) AS total_transactions
FROM `project-5400504384186300846.REPORT.DAILY_PAID_USER_TRANS_VER2`
WHERE TRANS_DATE BETWEEN '2026-01-01' AND '2026-01-31'
```
**Result:** 375,297,522 (Jan 2026)

### P2P Revenue (Actual Fees)
```sql
SELECT SUM(fee) AS total_revenue
FROM `momovn-prod.MBI_DA.HOANG_P2P_ALL_CASHOUT_W2W_BY_TID`
WHERE month = '2026-01-01'
```
**Result:** 49.4B VND (Jan 2026)

### Paylater MAU
```sql
SELECT COUNT(DISTINCT agent_id)
FROM `momovn-prod.BU_FI.PAYLATER_MAU_SEGMENT`
WHERE month_trans = '2026-01-01'
```
**Result:** 1,453,583 (Jan 2026)

### Vay Nhanh Disbursement
```sql
SELECT SUM(DISBURSED_AMOUNT)
FROM `momovn-prod.BU_FI.BAOTU_VAYNHANH_LOAN_INFO`
WHERE DISBURSED_DATE BETWEEN '2026-01-01' AND '2026-01-31'
```
**Result:** 3.1T VND (Jan 2026)

### FI Solutions (CLO) Disbursement
```sql
SELECT SUM(CONTRACT_AMOUNT)
FROM `momovn-prod.BU_FI.FIS_TCST`
WHERE SERVICE_TYPE = 'CLO'
  AND STATUS_CODE IN ('CONTRACT_SIGNED', 'DISBURSED')
  AND DATE_MODIFIED BETWEEN '2026-01-01' AND '2026-01-31'
```
**Result:** 124.6B VND (Jan 2026)

### DLS Online Transactions
```sql
SELECT COUNT(ORDER_ID)
FROM `momovn-prod.BU_ECOM.DLS_ALL_TRANS_V3_1`
WHERE DATE BETWEEN '2026-01-01' AND '2026-01-31'
```
**Result:** 113,509,211 (Jan 2026)

### TTT Users
```sql
SELECT IS_MP, COUNT(DISTINCT (CAST(REGEXP_EXTRACT(USER_ID, r'\d+') AS STRING))) AS total_users
FROM `momovn-prod.BU_FI.mart_ttt_daily_user_record`
WHERE GRASS_DATE >= '2026-01-01' AND GRASS_DATE <= '2026-01-31'
GROUP BY IS_MP
```
**Result:** Individual: 11.58M, Money Pool: 1.58M (Jan 2026)

### InsurTech Orders
```sql
SELECT COUNT(DISTINCT ID) AS total_successful_orders
FROM `momovn-prod.BU_FI.att_fi_ins_nl_ver2`
WHERE TRANS_DATE BETWEEN '2026-01-01' AND '2026-01-31'
  AND CC_STATUS = 'SUCCESS'
```
**Result:** 127,352 (Jan 2026)

### MiniApp Total Usage
```sql
SELECT SUM(total_records)
FROM `momovn-mini-app-analyst.MINI_APP_PLATFORM.STT_APDEX_BY_DETAIL`
WHERE report_date BETWEEN '2026-01-01' AND '2026-01-31'
```
**Result:** 1.57B uses (Jan 2026)

## Revenue vs Volume: Critical Distinction

| Domain | Returns | Type |
|--------|---------|------|
| P2P Revenue | `fee` column | **Actual revenue** |
| InsurTech | `FINAL_REV` column | **Actual revenue** |
| Vay Nhanh | `DISBURSED_AMOUNT` | Disbursement (proxy) |
| FI Solutions | `CONTRACT_AMOUNT` | Disbursement (proxy) |
| Paylater | `amount` | GMV (proxy) |
| DLS Online | `GMV` | GMV (proxy) |
| TTT | `BALANCE` / `cashin_gmv` | AUM/deposits (proxy) |
| Media AD | `cost` | **Cost** (what MoMo spends, not earns) |

Only P2P Revenue and InsurTech give actual fee/commission revenue. Everything else is a volume proxy. True P&L data requires FP&A sources beyond Mimir.
