# Eval Plan — mimir_set1 (2026-03-11)

## Goal

Evaluate the DA skill (me) by feeding it natural language questions only, having it generate SQL, then comparing results against ground truth.

## Source

`data/eval_sets/mimirset1/mimir_set1.csv` — 42 question/SQL pairs from domain owners.

## Steps

### Step 1: Extract tables from Expected_SQL

Parse each row's `Expected_SQL` column and extract all table references (e.g. `momovn-mimir.MIMIR.iDeA_TRANS_CORE`).

### Step 2: Check table access

For each unique table found in Step 1, run a dry-run query (e.g. `SELECT 1 FROM <table> LIMIT 0`) on the company network to check if our account (`son.pham9@mservice.com.vn`) via project `momovn-bu-fi-shared` has read access.

Result: a list of accessible vs. blocked tables.

### Step 3: Filter rows by access

Keep only the rows where ALL tables in their Expected_SQL are accessible. Drop the rest.

### Step 4: Run Expected_SQL for accessible rows

Execute each accessible row's Expected_SQL and capture the result set.

### Step 5: Produce 3 output files

| File | Contents | Purpose |
|------|----------|---------|
| `eval_input_questions_only.csv` | `index`, `Prompt`, `Domain` | Feed to DA — NO SQL, NO answers |
| `eval_ground_truth.csv` | `index`, `Prompt`, `Domain`, `Expected_SQL` | Reference for comparing SQL approaches |
| `eval_expected_results.csv` | `index`, `Prompt`, `Domain`, `Expected_SQL`, `Expected_Results` | Full ground truth with query results for scoring |

### Step 6: Run evaluation

For each question in `eval_input_questions_only.csv`:
1. DA receives only the natural language question
2. DA writes its own SQL
3. DA executes its SQL and gets results
4. Compare DA results vs. `eval_expected_results.csv` — match on data, not on SQL syntax

### Scoring

- **Exact match:** DA results == Expected results
- **Partial match:** Same structure, close numbers (within tolerance)
- **Miss:** Wrong data or query failure
